Research Proposal

Risk-Aware Hybrid Retrieval for Granite RAG

Project context: IBM Granite Needle-in-a-Haystack / Enterprise Retrieval Core theme: improving Granite RAG retrieval accuracy and reliability through hybrid retrieval and retrieval-side risk signals. Proposed title: Risk-Aware Hybrid Retrieval for Granite RAG

1. Research Motivation

Enterprise RAG systems search large document collections for rare, high-value evidence. In legal, compliance, healthcare, finance, and policy settings, an incorrect retrieved passage can lead to an incorrect answer even when the language model is strong.

Granite dense retrieval provides the semantic retrieval baseline for this project. It can retrieve paraphrased or conceptually similar evidence, while BM25 captures exact lexical details such as names, dates, numbers, clause-like expressions, and rare keywords. Combining these two retrieval modes gives a practical route to improving retrieval accuracy.

The remaining research problem is retrieval reliability. A hybrid retriever can still return evidence that looks plausible while being misleading: near-matching decoys, conflicting chunks, or low-confidence evidence. This proposal studies whether lightweight retrieval-side signals can predict these failure cases and guide corrective retrieval actions before Granite generates the final answer.

2. Paper Storyline

The paper follows a three-step progression:

StepSystem versionResearch role
V1Granite dense retrievalSemantic retrieval baseline.

The story is:

Granite dense retrieval provides the project’s semantic search foundation. Adding BM25 creates a hybrid retriever that preserves exact details and improves retrieval robustness. The hardest remaining cases involve retrieved evidence that appears relevant while carrying decoys, conflicts, or weak confidence. A retrieval reliability layer estimates these risks from the retrieved top-k chunks and triggers targeted actions such as stricter reranking, filtering, re-retrieval, warning, or abstention. The study evaluates this progression through focused baseline comparisons, risk prediction analysis, and risk-aware mitigation experiments.

3. Research Gap and Related Work

Existing retrieval and RAG work provides the background for each part of the proposal:

  • BEIR: retrieval baseline comparison for lexical, dense, and hybrid retrieval.
  • RAGAS: answer faithfulness and grounding evaluation.
  • Self-RAG, CRAG, Adaptive-RAG: adaptive and corrective RAG behaviour.
  • Conflicting-evidence RAG: disagreement between retrieved evidence.
  • Granite embedding models: IBM semantic retrieval foundation.

The project focuses on lightweight retrieval-side reliability signals for Granite-based enterprise needle retrieval. The signals are computed from retrieved evidence and require no newly trained controller.

4. Research Questions

RQ1: Hybrid retrieval improvement

How much does BM25 + Granite dense hybrid retrieval improve over Granite dense retrieval?

Purpose: test whether adding lexical retrieval improves the Granite semantic retrieval baseline.

Evidence: precision@1, recall@k, MRR, nDCG@10, and answer accuracy.

RQ2: Risk signal prediction

Can decoy-density, evidence conflict, and retrieval uncertainty predict retrieval or answer failures?

Purpose: test whether retrieval-side risk signals have diagnostic value.

Evidence: AUROC, calibration, and error rate across low/medium/high risk buckets.

RQ3: Risk-aware mitigation

Do risk-triggered actions reduce errors compared with fixed hybrid retrieval?

Purpose: test whether acting on risk signals improves high-risk cases.

Evidence: high-risk wrong-answer rate, answer accuracy, citation faithfulness, accuracy-coverage, and latency/cost.

5. Proposed Method

The proposed system is Risk-Aware Hybrid Retrieval for Granite RAG.

User query
  -> Granite dense retrieval
  -> BM25 lexical retrieval
  -> Hybrid merge / rank fusion
  -> Retrieval reliability signals
  -> Risk-aware evidence action
  -> Granite answer generation with citations

5.1 Granite Dense Retrieval Baseline

Granite dense retrieval embeds the user query and document chunks into a semantic vector space, then retrieves chunks with high semantic similarity. This is the baseline system for the retrieval study.

5.2 Hybrid Retrieval

BM25 retrieves exact lexical matches. Granite dense retrieval retrieves semantically similar chunks. The hybrid retriever merges both ranked lists using score fusion or rank fusion. This stage tests whether lexical matching improves Granite semantic retrieval, especially for exact entities, numbers, dates, and rare keywords.

5.3 Retrieval Reliability Signals

SignalComputed fromPredictsAction
Decoy-densityTop-k chunks with high similarity and key entity/date/number/condition/negation mismatchWrong top-1 evidence or wrong answerStricter rerank/filter
Evidence conflictChunks implying incompatible answer candidates, versions, or conditionsMixed or contradictory answerGroup by version, condition, source, or answer candidate
Retrieval uncertaintyScore margin, BM25-dense disagreement, top-k entropy/diversityLow evidence confidenceRe-retrieve, warn, or abstain

5.4 Risk-Aware Actions

Risk profileAction
Low riskUse the hybrid top-ranked evidence.
High decoy-densityApply stricter reranking or filtering around entities, dates, numbers, and negation.
High conflictSeparate evidence groups and select evidence matching the query condition.
High uncertaintyRe-retrieve, rewrite the query, warn, or abstain.

7. Data and Benchmark Design

The evaluation uses two complementary data sources.

Public Retrieval Benchmark

Use SciFact or a small BEIR-style subset to evaluate general retrieval quality. This benchmark supports the comparison between Granite dense retrieval, BM25, and hybrid retrieval using standard retrieval metrics.

Controlled Synthetic Needle Dataset

The controlled synthetic dataset evaluates the core research contribution. It creates hard retrieval cases where the true evidence is surrounded by near-matching or conflicting evidence.

Difficulty factors:

  • Decoy count: number of misleading near-matches.
  • Decoy similarity: lexical and semantic closeness to the true evidence.
  • Mismatch type: entity, number, date, condition, or negation.
  • Conflict type: old vs new policy, general rule vs exception, source A vs source B.

This dataset supports the risk prediction and risk-aware mitigation experiments.

Retrieval quality:

  • answer accuracy;
  • wrong-answer rate;
  • citation faithfulness.

Risk reliability:

  • AUROC for failure prediction;
  • calibration by risk bucket;
  • high-risk subset accuracy;
  • accuracy-coverage curve for warning or abstention.

Efficiency:

  • retrieval latency;
  • reranking cost;
  • number of extra retrieval attempts.

9. Expected Contributions

The proposal has four contributions:

  1. Granite RAG delivery system: a working system that retrieves evidence, generates answers, and shows source citations.
  2. Hybrid retrieval evaluation: a focused comparison testing whether BM25 improves Granite dense retrieval.
  3. Retrieval reliability layer: decoy-density, evidence conflict, and uncertainty signals computed from top-k retrieved chunks.
  4. Controlled failure evaluation: a synthetic needle dataset and high-risk subset analysis for testing failure prediction and mitigation.

The central research contribution is the retrieval reliability layer and its evaluation. Hybrid retrieval is the first accuracy improvement step that supports the full system.

11. Abstract Draft

This project investigates risk-aware hybrid retrieval for IBM Granite RAG in enterprise needle-in-a-haystack settings. The system starts from Granite dense retrieval, adds BM25 lexical retrieval to form a hybrid retriever, then applies a lightweight retrieval reliability layer that estimates decoy-density, evidence conflict, and retrieval uncertainty from retrieved top-k chunks. The evaluation first tests whether hybrid retrieval improves over Granite dense retrieval, then tests whether retrieval-side risk signals predict retrieval and answer failures, and finally tests whether risk-triggered actions reduce high-risk errors. Public retrieval benchmarks establish general retrieval performance, while a controlled synthetic needle dataset evaluates near-matching decoys and conflicting evidence. The expected contribution is a working Granite RAG system and a focused study of retrieval-side reliability signals for improving high-risk RAG behaviour.

12. Final Framing

The project delivers a working Granite RAG system and studies retrieval reliability. The first accuracy step adds BM25 lexical retrieval to Granite dense retrieval to form a hybrid retriever. The research contribution is a retrieval reliability layer that detects decoys, conflicting evidence, and uncertainty, then uses those signals to guide evidence selection and reduce high-risk errors.
